# AGENTS.md

このリポジトリはハンズオン用です。Codex は、ユーザーが自分で実装して学ぶ前提を壊さないように振る舞います。

## 基本方針

- ユーザーが明示的に依頼しない限り、ソースコードを編集しない。
- ユーザーが明示的に依頼しない限り、`git add`、commit、push をしない。
- 「終わった気がする」「見て」「確認して」などの依頼では、テスト実行、差分確認、問題点の説明に留める。
- 修正案を出す場合は、原則としてユーザーが自分で直せるように、対象ファイル・考え方・確認コマンドを示す。
- どうしても編集が必要だと判断した場合でも、先に理由と編集範囲を説明し、ユーザーの明示的な許可を待つ。

## ハンズオン中のレビュー運用

- 章ごとの完了確認では、対応する `scripts/check-chapter.sh <章番号>` と必要最小限の `go test` を実行する。
- 未実装箇所や TODO が残っている場合は、勝手に消さず、残っている場所と意味を説明する。
- staged / unstaged / untracked の状態は報告するだけにする。ユーザーの明示的な依頼なしに staging 状態を変更しない。

## ブランチ・commit・push

- ハンズオンの通常運用では、Codex はブランチを作らない。
- ハンズオンの通常運用では、Codex は commit しない。
- ハンズオンの通常運用では、Codex は push しない。
- ブランチ作成、commit、push は、ユーザーが明示的に依頼した場合だけ行う。
- push はリモートに公開される操作なので、実行前に対象ブランチ、commit 内容、push 先を確認する。

## 例外

- ユーザーが「実装して」「直して」「このファイルを更新して」と明示した場合は、指定範囲内で編集してよい。
- ユーザーが「commit して」「push して」と明示した場合だけ、git 操作を行ってよい。
- それでも、破壊的操作や既存変更の取り消しは、必ず事前に確認する。
